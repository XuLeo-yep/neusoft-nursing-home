# 东软颐养中心管理系统 — 设计文档

## 一、项目概述

基于 Spring Boot + Vue 3 的养老院综合管理系统，涵盖客户管理、床位管理、护理管理、外出/退住审批四大核心业务模块。

### 技术栈

| 层 | 技术 | 版本 |
|---|------|------|
| 后端框架 | Spring Boot | 4.0.6 |
| ORM | MyBatis-Plus | 3.5.15 |
| 鉴权 | Sa-Token | 1.45.0 |
| 数据库 | MySQL | 8.0 |
| 前端框架 | Vue 3 + Element Plus | |
| 构建工具 | Vite | 5.x |
| 状态管理 | Pinia | |
| HTTP 客户端 | Axios | |

### 部署架构

```
┌──────────┐    HTTP/api    ┌──────────────┐    JDBC    ┌─────────┐
│  浏览器   │ ←──── Proxy ──→ │  Spring Boot │ ←────────→ │  MySQL  │
│ :3000    │               │  :8088       │            │ :3306   │
│ Vue 3    │               │  REST API    │            │ 12张表  │
└──────────┘               └──────────────┘            └─────────┘
```

## 五、后端分层架构

```
controller/   ── @RestController, 接收 HTTP 请求, 返回 JSON
    │ @Autowired
service/      ── @Service, 业务逻辑
    │ @Autowired
mapper/       ── @Mapper extends BaseMapper<T>, 数据访问, 自动 CRUD
    │ MyBatis-Plus
entity/       ── @TableName, 数据实体, 映射数据库表
    │ JDBC
  MySQL       ── 12 张表
```

**辅助层**：
- `config/SaTokenConfig` — 登录拦截、权限校验
- `common/Result` — 统一响应体 `{code, msg, data}`
- `handler/GlobalExceptionHandler` — 全局异常 → JSON

## 七、业务流程

### 7.1 登录流程

```
用户输入账号密码 → POST /api/user/login
  → UserMapper.selectByLoginCode(loginCode)
  → 验证密码 → StpUtil.login(userId)
  → 返回 {token, role, username, userId}
  → 前端存 Pinia + localStorage
  → 跳转 Dashboard
```

### 7.2 入住登记流程

```
填写客户信息 → POST /api/customer/add
  → CustomerMapper.insert(customer)
  → 创建床位使用记录
  → 返回成功
```

### 7.3 退住审批流程

```
健康管家提交退住申请 → POST /api/application/checkout/submit
  → CheckOutApplicationMapper.insert(申请，状态=待审批)
管理员查看待审批 → GET /api/application/checkout/list
  → 批准/拒绝 → PUT /api/application/checkout/{id}/approve
  → 更新申请状态 + 客户状态 → 计算退费
```

### 7.4 日常护理流程

```
设置客户护理级别 → POST /api/nursing/customer/{cid}/level/{lid}
购买护理项目 → POST /api/nursing/customer/{cid}/project
执行护理 → POST /api/nursing/record/add
查看记录 → GET /api/nursing/record/list/{cid}
```

## 八、角色权限

| 角色 | 权限范围 |
|------|---------|
| admin | 全部权限：用户管理、客户CRUD、床位管理、护理配置、审批 |
| health_worker | 受限：查询客户、日常护理、提交外出/退住申请 |

权限通过 `Role` 枚举定义，`Permission` 枚举管理具体操作点，后续通过 Sa-Token `@SaCheckPermission` 注解控制。
